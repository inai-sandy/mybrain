For the curious — the real engine
The agent runs on LangGraph.js — an open-source (MIT) library that runs inside our own app and writes every step to our own SQLite. So a run survives a server restart and resumes exactly where it paused — we tested this: the process was killed mid-run and picked up clean from the saved checkpoint. The thinking is done by the Codex CLI running headless on a flat ChatGPT subscription (no per-use bill), with Claude Sonnet as an automatic fallback if the daily limit is hit. Connectors live in our own app with secrets encrypted at rest, and are offered to the agent over one in-house MCP server — the secrets themselves never reach the model.

For the curious — how the memory loop works
Before a run, the agent searches your second brain (the memory vault + SuperMemory) for anything relevant and shows it — you can drop a wrong item before it proceeds. After a run, it proposes new facts; only the ones you keep are written back, each tagged with the agent and the exact run so you can trace and reverse them. Every agent also keeps its own little notebook that carries across its recurring runs. Your Vault and secrets are never read into an agent.
